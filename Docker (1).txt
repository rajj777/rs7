Q3. Dockerfile – Java Application

1.Write a simple Java program that prints a message.
->vim Hello.java   <--- this file will look like

public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello from Dockerized Java!");
    }
}

2.Write a Dockerfile
->vim Dockerfile    <--- in that file

# Use an official JDK image to compile the program
FROM openjdk:17-slim AS build

# Set working directory
WORKDIR /app

# Copy the Java source file
COPY Hello.java .

# Compile the Java program
RUN javac Hello.java

# Use a smaller JRE image for running
FROM openjdk:17-slim

WORKDIR /app

# Copy compiled class file from build stage
COPY --from=build /app/Hello.class .

# Command to run the program
CMD ["java", "Hello"]

3.Build the Docker image
->docker build -t java-app .

4.Run the container
->docker run --rm java-app  <--it will run Hello.java

Q6. Bind Mount vs Volume

1.Create a directory on the host system and store a file in it.
->mkdir ~/mydata
->echo "Hello from host bind mount" > ~/mydata/hostfile.txt

2.Run a container with a bind mount:
->docker run -it --name bind_test -v ~/mydata:/data ubuntu

3.Inside the container, check:
->ls /data
->cat /data/hostfile.txt

4.Compare this with using a Docker volume and explain the difference.
Bind mounts: Directly expose host files/directories inside a container. 
Great for development, but tightly coupled to host paths.
Volumes: Docker-managed storage that persists independently of containers 
and host paths. Ideal for production and stateful apps.

Q4. Docker Swarm: Rolling Update

1.Deploy a service using image nginx:1.25
->docker service create --name nginx-service --replicas 3 -p 8080:80 nginx:1.25

2.Update the service to use image nginx:latest
Use rolling update options:
Update one container at a time
Delay between updates: 10 seconds
->docker service update \
  --image nginx:latest \
  --update-parallelism 1 \
  --update-delay 10s \
  nginx-service

3.Verify update status.
->docker service ls
->docker service ps nginx-service
->docker service inspect --pretty nginx-service

4.Roll back if needed
->docker service update --rollback nginx-service
